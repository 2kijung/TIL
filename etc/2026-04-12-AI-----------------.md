# [etc] AI 비서 자동 포스팅 시스템 구축

> 작성일: 2026-04-12

## 📝 내용

## 목표
Mac Mini M4에서 공부 내용을 GitHub, Notion, Velog, Tistory에 한 번에 자동 포스팅하는 시스템 구축.

## GitHub ✅ 성공
**방법:** GitHub REST API (PUT /contents)로 마크다운 파일 직접 커밋
**막힌 점:** 한글 파일명 URL 인코딩 오류 (`ascii codec can't encode`)
**해결:** `urllib.parse.quote()`로 파일명 URL 인코딩 처리 + safe_title에서 한글 제거

## Notion ✅ 성공
**방법:** Notion Integration 생성 → API 토큰 발급 → 데이터베이스 연동
**구현:** `POST /v1/pages` 로 Title, Date, Category 속성 포함 페이지 자동 생성

## Velog ✅ 성공 (우회 방법 포함)
**처음 시도:** GraphQL API (`v3.velog.io/graphql`) mutation → Cloudflare가 외부 클라이언트 차단, 빈 응답 반환
**두 번째 시도:** Playwright headless 브라우저 + playwright-stealth → Cloudflare Turnstile이 headless 감지, Verification failed
**최종 해결:** `undetected-chromedriver` 라이브러리 사용
- headless=False 모드로 실제 Chrome 실행
- 쿠키로 로그인 상태 주입 (access_token, refresh_token)
- CodeMirror 에디터는 JS `setValue()`로 직접 본문 입력
- `[data-testid='publish']` 버튼 클릭으로 출간

## Tistory ⚠️ 진행 중
**현재 상태:** undetected-chromedriver + TinyMCE iframe 에디터 조작까지 구현
- TSSESSION 쿠키로 로그인 → 글쓰기 → 제목/본문/태그 입력 → 완료 버튼 → 공개 발행 버튼까지 동작
- 공개 발행 후 로그인 페이지로 튕기는 현상 (쿠키 도메인 범위 이슈)
**내일 해결:** .tistory.com 전체 도메인으로 쿠키 설정 방식 수정

## 사용 스택
- Python 3.11, undetected-chromedriver (Selenium), GitHub API, Notion API
- OpenClaw AI 에이전트 프레임워크 (Telegram 연동)
- Mac Mini M4 (Apple Silicon, macOS)
