## 🔥 현재 상태 진단

Lv1 문제를 풀면서 느낀 점

- 단순 구현은 가능
- 하지만 **비효율적인 접근 (문자열 남발)** 존재
- 자료구조 활용(ArrayList 등) 부족
- 대신 **그리디 감각은 올라오는 중**

👉 결론: **Lv1 중반 → 후반 진입 단계**

## 📌 핵심 패턴 정리

### 1️⃣ 숫자 처리 (문자열 ❌ → 수학 ⭕)

❌ 잘못된 방식
```java
String str = String.valueOf(n);
String[] arr = str.split("");
```

⭕ 정석 방식
```java
while(n > 0){
    int digit = n % 10;
    n /= 10;
}
```
👉 자릿수 문제는 무조건 `%10`, `/10`으로 처리

---

### 2️⃣ 문자열 처리 기본
```java
for(char c : s.toCharArray()){
    // 문자 하나씩 처리
}
```
👉 문자열 문제 80%는 이걸로 해결 가능

---

### 3️⃣ 문자열 뒤집기
```java
StringBuilder sb = new StringBuilder(str);
sb.reverse();
return sb.toString();
```
👉 reverse는 그냥 외워라

---

### 4️⃣ 정렬
```java
char[] arr = String.valueOf(n).toCharArray();
Arrays.sort(arr);
```
👉 문자열 → char 배열 → 정렬

---

### 5️⃣ 부분 문자열
```java
s.substring(start, end)
```
👉 인덱스 감각 중요 (start 포함, end 미포함)

---

### 6️⃣ 그리디 (예산 문제)
```java
Arrays.sort(arr);
for(int i = 0; i < arr.length; i++){
    if(arr[i] <= budget){
        budget -= arr[i];
        count++;
    } else {
        break;
    }
}
```
👉 **정렬 후 작은 것부터 선택 = 그리디 기본 패턴**

---

### 7️⃣ 최대공약수 (유클리드 호제법)
```java
while(b != 0){
    int tmp = a % b;
    a = b;
    b = tmp;
}
```
👉 무조건 암기

## 🚨 실수 & 깨달은 점

### ❌ 1. 문자열 남발
자릿수 더하기, 하샤드 수를 문자열로 처리
→ **해결: 수학적으로 처리 (%10)**

### ❌ 2. 배열에 push 시도
```java
answer.push(...)  // 자바는 안됨
```
→ 크기 알면 **배열**, 모르면 **ArrayList**

### ❌ 3. String += 반복
```java
answer += "문자";  // 성능 안좋음
```
→ `StringBuilder` 사용
```java
StringBuilder sb = new StringBuilder();
sb.append(...);
```

### ❌ 4. double 비교 실수
```java
double root = Math.sqrt(n);
if(root * root == n)  // 위험
```
→ **해결**
```java
long root = (long)Math.sqrt(n);
if(root * root == n)  // 안전
```
👉 **제곱근은 double → 검증은 long**

## 💡 대표 문제별 핵심 요약

| 문제 유형 | 핵심 방법 |
|---|---|
| 자릿수 더하기 | `%10`, `/10` |
| 자연수 뒤집기 | 배열 + `%10` |
| 정렬 문제 | `Arrays.sort()` + reverse |
| 문자열 문제 | `toCharArray()` |
| 구간 합 | `(max - min + 1) * (min + max) / 2` |
