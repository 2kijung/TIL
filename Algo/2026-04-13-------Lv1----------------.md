단순 풀이 암기가 아니라 **"유형 파악 → 실수 원인 → 해결 방법"** 흐름으로 정리한다.

## 1. 문자열 → 정수 변환

**유형:** 기본 타입 변환

❌ 실수
```java
Integer.ParseInt(s); // 대문자 P → 컴파일 에러
```

✅ 해결
```java
Integer.parseInt(s); // 자바는 camelCase, p 소문자
```

---

## 2. 약수의 합

**유형:** 완전탐색 (for문 + 조건문)

❌ 실수
```java
if(i % n == 0) // 반대로 씀
```

✅ 해결
```java
if(n % i == 0) // 약수 조건: n이 i로 나눠져야 함
```

핵심: "누가 누구로 나눠지는지" 방향을 정확히 이해할 것

---

## 3. 자릿수 더하기

**유형:** 숫자 자릿수 분해

❌ 처음 접근 (문자열 방식 - 비효율)
```java
String str = String.valueOf(n);
for(String s : str.split("")) answer += Integer.parseInt(s);
```

✅ 더 좋은 방법 (수학 방식)
```java
while(n > 0) {
    answer += n % 10; // 마지막 자리 추출
    n /= 10;          // 자리 제거
}
```

핵심: 숫자 문제는 문자열보다 수학이 더 효율적

---

## 4. 자연수 뒤집어 배열로 만들기

**유형:** 자릿수 분해 + 배열 저장

❌ 실수
```java
int[] answer = {};
answer.push(n % 10); // 배열에 push 없음, 크기 미지정
```

✅ 해결
```java
int len = String.valueOf(n).length();
int[] answer = new int[len];
int i = 0;
while(n > 0) {
    answer[i++] = (int)(n % 10);
    n /= 10;
}
```

핵심: 배열은 크기 고정 + index로 접근, `answer[i++]` 패턴 기억

---

## 5. 정수 내림차순 배치

**유형:** 문자열 변환 + 정렬

❌ 실수
```java
strArr.sort(); // 자바 배열에 sort() 메서드 없음
```

✅ 해결
```java
char[] arr = String.valueOf(n).toCharArray();
Arrays.sort(arr);
StringBuilder sb = new StringBuilder(new String(arr));
sb.reverse();
return Long.parseLong(sb.toString());
```

핵심 흐름: 숫자 → 문자 배열 → Arrays.sort() → reverse → 다시 숫자

---

## 🔥 오늘 반복된 실수 패턴

| 실수 | 원인 | 해결 |
|------|------|------|
| ParseInt vs parseInt | camelCase 미숙 | 자바는 소문자 시작 |
| i % n vs n % i | 방향 혼동 | 약수 조건은 n % i |
| 배열에 push | 언어 혼동 | 배열은 index, 리스트는 add |
| 문자열로만 풀기 | 습관 | 숫자 문제는 수학 방식 먼저 고려 |

---

## 📌 오늘 익힌 핵심 패턴

**자릿수 분해**
```java
n % 10  // 마지막 자리 추출
n /= 10 // 자리 제거
```

**정렬**
```java
Arrays.sort(arr);         // 오름차순
new StringBuilder(str).reverse(); // 내림차순
```

**배열 vs 리스트**
- 배열: 크기 고정, index 접근
- 리스트: 크기 가변, add() 사용
