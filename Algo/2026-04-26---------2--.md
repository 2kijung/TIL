# 📚 코딩테스트 Lv1 핵심 정리 (실전용 완성본)

코딩테스트를 준비하면서 자주 나오는 패턴과 핵심 포인트를 정리했다.
Lv1 기준으로 빠르게 풀기 위한 사고 방식 + 실전 코드 패턴 중심이다.

## 1. 문자열 처리

### 숫자로만 이루어졌는지 판별
for (char c : s.toCharArray()) {
    if (!Character.isDigit(c)) return false;
}
핵심: 예외 처리(parseInt) 말고 문자 단위 검사. 코테 정석 패턴.

### toCharArray() 이해
String str = "18"; → char[] arr = str.toCharArray(); → [1, 8]
for(char c : str.toCharArray()){ sum += c - 0; }
핵심: 문자열을 한 글자씩 쪼갬. 1 → 숫자 1로 바꾸려면 c - 0

### 문자열 일부 추출
s.substring(start, end)

### 문자열 반복
"*".repeat(n)

## 2. 문자열 정렬
char[] arr = s.toCharArray();
Arrays.sort(arr);
return new StringBuilder(new String(arr)).reverse().toString();

## 3. 숫자 처리 (%10 패턴 핵심)
while (n > 0) { sum += n % 10; n /= 10; }
자릿수 합, 숫자 뒤집기, 자리수 분리 → 문자열보다 이 방식이 코테 정석

## 4. 수학 최적화 (반복문 제거)
long total = (long) price * count * (count + 1) / 2;

## 5. 자료형 (오버플로우 방지 중요)
- int → 약 21억 / long → 그 이상
- 10,000,000 * 1000 = 10,000,000,000 → int 초과 → long 필수
sum += (long)x * i; → 계산 시작부터 long. 마지막 캐스팅은 의미 없음.

## 6. 배열 / 리스트 연속 중복 제거
if(list.isEmpty() || list.get(list.size() - 1) != num) { list.add(num); }

## 7. 배열 생성
long[] arr = new long[n]; → 배열은 반드시 크기 지정. long[] answer = {}; 안됨.

## 8. 반복문 실수 방지
for(int i = 0; i < n; i++) → i = n 아님, i < n

## 9. x 간격 n개 숫자
for (int i = 0; i < n; i++) { answer[i] = (long)x * (i + 1); }
i는 0부터 → (i + 1). long 캐스팅 필수.

## 10. 나머지 문제 접근
n % x == 1 패턴:
for (int x = 2; ; x++) { if (n % x == 1) return x; }
조건 그대로 구현. x=1은 의미 없음.
수학적 변환: n % x == 1 → n - 1 = x의 배수

## 16. 투포인터 (Lv2 핵심)
while (right < arr.length) {
    sum += arr[right];
    while (sum > target) { sum -= arr[left++]; }
    if (sum == target) answer++;
    right++;
}

## 17. 해시

## 18. 스택

## 19. 제곱근 / 완전제곱수 판별
long x = (long)Math.sqrt(n);
if (x * x == n) { return (x + 1) * (x + 1); }
return -1;
sqrt는 double 반환 → long으로 검증 필수

## 20. 짝수 / 홀수 판별
return num % 2 == 0 ? "Even" : "Odd";

## 21. 평균 구하기
double sum = 0;
for (int n : arr) { sum += n; }
return sum / arr.length; → 반드시 double로 계산

## 최종 알고리즘 요약
- 문자열 → substring / repeat / toCharArray
- 숫자 → %10
- 정렬 → sort + reverse
- 수학 → 공식
- 배열 → 이전값 비교 / 크기 선언
- 반복문 → i < n
- 자료형 → long
- 탐색 → HashMap / Set
- 구간 → 투포인터
- 구조 → Stack
- 패턴 → % / repeat

## 실전 사고 방식
1. 이거 문자열 문제인가? 숫자 문제인가?
2. 숫자면 %10 떠올리기
3. 값 크면 long 체크
4. 조건 그대로 구현

이 구조로 계속 쌓으면 단순 암기가 아니라 패턴 자동 반응 단계까지 올라간다.
